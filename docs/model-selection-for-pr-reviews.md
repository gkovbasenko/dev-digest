# Вибір core-моделі для AI-рев'ю пул-реквестів (DevDigest)

> **Статус:** чернетка-рішення (decision doc) · **Дата:** 2026-06-05
> **Контекст:** вибір LLM для агента-рев'юера, що запускатиметься в CI на кожен PR
> через OpenRouter (один ключ → багато моделей).
>
> ⚠️ **Ціни й бенчмарки волатильні** (моделі/тарифи змінюються щомісяця). Усі
> числа тут — станом на червень 2026; перед фіналізацією звіряти на живих
> сторінках провайдерів. Числа вартості — **оцінки** на основі реального заміру
> токенів (див. §4); токенайзери різних моделей відрізняються на ±10–15%.

---

## 1. Мета й обмеження

- **Дешево** — рев'ювер крутиться на КОЖЕН PR, вартість має бути близька до нуля.
- **Якість ≥ Claude Sonnet 4.6** — наша планка для пошуку логічних/безпекових багів.
- **Reasoning бажано** — PR-рев'ю виграє від «думання» (subtle bugs), але воно не
  має роздувати вартість.
- **Один шлюз** — OpenRouter: один `OPENROUTER_API_KEY` у Secrets репозиторію,
  доступ до всіх моделей, перемикання per-agent без зміни інфраструктури.
- **Ключ — у GitHub *Secrets*, не Variables** (Variables світяться в логах).

## 2. Архітектурні знахідки з нашого коду (впливають на вартість)

- `reviews/service.ts` робить: `assemblePrompt(system + skills + memory + specs + diff)`
  → (map-reduce за потреби) → `llm.completeStructured({schema: Review})`
  → `groundFindings` (citation-gate, **чистий, без БД**) → постинг.
- **`FILE_MAP_THRESHOLD_LINES = 400`** (`reviews/constants.ts`): diff >400 рядків і >1
  файл → **map-reduce по файлах** (один LLM-виклик на файл). Це **token-limit
  resilience** з епохи моделей на 8–32K контексту, **не оптимізація вартості**.
- При map-reduce фіксований контекст (system+skills+memory+specs ≈ harness)
  **повторюється N разів** (по разу на файл) → дорого.
- **Окремий intent-pass** (`deriveIntent`) шле **весь diff ще раз** → ~подвоює
  вхідні токени.
- LLM-клієнт — `OpenAIProvider` (`new OpenAI({apiKey})`). OpenRouter OpenAI-сумісний
  → достатньо додати `baseURL: 'https://openrouter.ai/api/v1'`. Claude Code SDK
  **не потрібен** (він приходить лише з агент-ранером claude-code-action).
- `structured.ts` має `parseWithRepair` (reprompt-on-error) → страхує моделі без
  strict `json_schema`.
- `pricing.ts` **не знає DeepSeek/GLM/MiniMax/Qwen** → `estimateCost` поверне `null`.
  Треба додати рядки, щоб трейси рахували реальні $.

## 3. Модель білінгу: підписка проти API

| | Підписка (Claude Code / GLM Coding Plan) | API (pay-per-token) |
|---|---|---|
| Як платиш | фіксована $/міс, rate-ліміти | prepaid-баланс, за токени |
| CI/автоматизація | ⚠️ зазвичай **заборонено/не призначено** | ✅ дозволений шлях |
| Підходить для | інтерактивна робота людини в IDE/CLI | боти, CI, unattended |

**Висновок:** для «рев'юера на кожен PR» правильний (і часто єдино дозволений)
шлях — **API pay-per-token**, не підписка.

- **Claude Code Pro/Max ($20–200/міс):** ToS Anthropic **прямо забороняє** CI/CD.
  Плюс rate-ліміти (Pro ≈ 44k ток/5год) вибиваються одним PR. → не варіант.
- **z.ai GLM Coding Plan ($10/$30/$80 на міс, білінг поквартально):** бандл доступу
  до GLM для coding-тулів (Claude Code, Cline…). Призначення — інтерактивне;
  **придатність для CI треба звірити в їхньому ToS** (не підтверджено). За
  замовчуванням орієнтуємось на per-token API.

### 3.1 OpenRouter проти прямого підключення до провайдера

**OpenRouter — не дискаунтер, а роутер.** Він проксіює ціну базового провайдера
~один-в-один, а заробляє на **~5% комісії при поповненні балансу**:

```
ціна через OpenRouter ≈ ціна провайдера + ~5% (на поповненні)
```

Тож **напряму до провайдера** (DeepSeek, z.ai) може бути трохи дешевше за рахунок:
1. немає ~5% комісії OpenRouter;
2. **нативного кешування** першоджерела (DeepSeek auto context-cache: вхід cache-hit
   ~$0.028/M замість $0.14; z.ai «Cached Input»: GLM-5.1 $0.26 замість $1.4) —
   часто більший важіль, ніж 5%;
3. подеколи off-peak знижок.

**Але:** OpenRouter дає **один ключ/баланс на всі моделі + failover + вибір
найдешевшого хоста** (DeepSeek/GLM/Qwen — open-weight, їх хостять Together,
Fireworks, DeepInfra, Novita… за різними цінами). Для дизайну «модель per-agent»
це операційно набагато простіше, ніж N акаунтів/ключів/балансів.

| | OpenRouter | Напряму (DeepSeek/z.ai) |
|---|---|---|
| Ціна за токен | провайдер + ~5% | провайдер (− кеш) |
| Ключі / акаунти | **один** на всі моделі | по одному на провайдера |
| Failover / вибір хоста | ✅ авто | ручний |
| Кешування | залежить від хоста | ✅ нативне |

**Reality-check (наш масштаб).** Одне рев'ю на DeepSeek V4 Flash ≈ $0.009 → 5% =
$0.0005. Навіть 100 PR/міс: економія прямого підключення ≈ **$0.05/міс**. Кеш теж
дає мало, бо в single-pass домінує **сам diff (50k ток, унікальний, не кешується)**,
а harness (~3.6k) — мала частка. Реальні важелі вартості (на порядки більші):
single-pass vs map-reduce (×4–10), обсяг reasoning, виключення boilerplate.

**Рішення:** поки **обираємо модель — сидимо на OpenRouter** (гнучкість > 5%).
Після **фіксації моделі** + зростання обсягу — розглянути пряме підключення до того
провайдера заради 5% + нативного кешу. Передчасно оптимізувати роутинг на 5%, коли
single-pass зрізає ~75%, — не варто.

## 4. Виміряний baseline токенів (реальний diff)

Заміряно `tiktoken` (o200k_base) на реальному diff репозиторію:

- **diff:** 3 827 рядків / 153 KB → **50 868 токенів** (~3.0 символи/токен — код щільний)
- **файлів:** 47 (один lock-файл = **20 951 ток = 41% усього diff!**)
- **harness (FC):** system 575 + injection-guard 37 + skills/memory/specs/task ~3000 ≈ **3 612 ток**
- **single-pass вхід:** diff + FC = **54 480 ток**; вихід рев'ю ~1.5k (+ reasoning-токени)

## 5. Стратегія: single-pass, НЕ map-reduce

На 47-файловому diff map-reduce повторює harness 47× → вхід **220k** проти **54k**
у single-pass (×4). А diff (50k) **влазить із запасом** у вікно сучасних дешевих
моделей (DeepSeek V4 Flash / Qwen3.6 — 1M; GLM-5.1 — 203K).

### ⚠️ Map-reduce + reasoning = катастрофа

Reasoning «думає» в **output**-токенах. У map-reduce кожен із 47 викликів думає
окремо → output вибухає (~112k ток). Виміряно (4000-рядковий PR):

| Модель | single-pass | map-reduce (47 викликів) | різниця |
|---|---|---|---|
| DeepSeek V4 Flash | $0.009 | $0.063 | ×7 |
| MiniMax M2.5 | $0.024 | $0.20 | ×8 |
| Qwen3.6 Plus | $0.030 | $0.29 | ×10 |

**Рішення:** для reasoning-моделей single-pass **обов'язковий**. Map-reduce лишити
тільки як фолбек, коли diff не влазить у вікно — і тоді **bin-packing** (2–3 великі
чанки), а не виклик-на-файл.

### Механіка вартості reasoning

`вартість думання = (токени думання) × (ціна output)`. Тому «дешевий reasoning» =
reasoning-модель **з дешевим output** (V4 Flash $0.28, GLM-4.7-FlashX $0.4), а не з
дорогим (R1 $2.50, Sonnet $15).

## 7. Аналіз z.ai (Zhipu AI / GLM) — відповіді на два питання

**(a) Доступ до API.** z.ai має **два окремі продукти** — не плутати:
- **Z.AI API Platform — pay-per-token** (це і є потрібний нам «API-план»). Окремий
  API-ключ + prepaid-баланс; ціни — таблиця §6 / сторінка Pricing (GLM-5.1 $1.4/$4.4,
  GLM-4.7 $0.6/$2.2, GLM-4.7-FlashX $0.07/$0.4 …). OpenAI-сумісний ендпоінт; ці ж
  моделі доступні і через OpenRouter. **Це наш шлях для CI.**
- **GLM Coding Plan — підписка** (Lite ~$10 / Pro ~$30 / Max ~$80 на міс, поквартально),
  для інтерактивних coding-тулів (Claude Code, Cline…). **Не для CI** (звірити ToS) —
  нам не потрібна.
- Плюс **безкоштовний тариф**: GLM-4.7-Flash, GLM-4.5-Flash — free (rate-limited),
  для прототипу/евалів.

Два способи дотягнутись до GLM з рев'юера: (1) через **OpenRouter** (наш дефолт —
один ключ на всі моделі, +~5% комісія на поповнення); (2) **напряму z.ai API**
(окремий ключ/баланс, без націнки, OpenAI-сумісний base URL).

**(b) Чи їхні моделі кращі за наші три?** z.ai додає **дві цінні опції на обох
кінцях спектра**:
- **GLM-5.1 — топ якості.** SWE-bench **Pro #1 SOTA (70%)**, обходить Sonnet на Pro,
  open-weight (MIT). ~$0.105/рев'ю — дорожче за MiniMax, але **у ~2.5× дешевше за
  Sonnet** і потенційно якісніше. Кандидат на «преміум-агента».
- **GLM-4.7-FlashX / GLM-4.7-Flash — ультра-дешево.** FlashX ($0.07/$0.4) ≈
  **$0.006/рев'ю** (дешевше за DeepSeek V4 Flash на вході), а Flash — **безкоштовний**
  (rate-limited) для прототипування/евалів.
- GLM-4.7 ($0.6/$2.2, 73.8% Verified) — між ними, нічим не кращий за MiniMax M2.5.

## 8. Рекомендація

1. **Дефолт (дешево + reasoning майже безкоштовний):** **DeepSeek V4 Flash** —
   ~$0.01/PR, 1M контексту (single-pass завжди влазить), перемикач thinking on/off.
   Альтернатива на цьому ж рівні цін: **GLM-4.7-FlashX** (~$0.006).
2. **Доведена якість ≥ Sonnet за копійки:** **MiniMax M2.5** — 80.2% Verified
   (>Sonnet), ~$0.024/PR (~11× дешевше за Sonnet).
3. **Преміум-агент (найвища якість):** **GLM-5.1** — #1 на SWE-bench Pro, ~$0.105/PR
   (~2.5× дешевше за Sonnet). Вмикати на критичні репозиторії через той самий ключ.
4. **Безкоштовний baseline для евалів:** **GLM-4.7-Flash** ($0).

Стратегія однакова для всіх: **single-pass + token-budgeter (pre-count → вибір
вікна → bin-pack лише за потреби) + виключення boilerplate + intent згорнути в той
самий виклик**.
